Project Overview & Problem Statement

Problem Statement

Students experience massive academic, social, and psychological pressure. Standard stress indicators are often missed until they manifest as performance drops or severe mental strain. Proactive, machine-learning-based modeling is needed to detect high stress indices before critical symptoms arise.

Project Objectives

  • Proactive Modeling: Create a predictive backend framework that determines stress triggers instantly.
  • Categorization: Group stress levels into actionable categories (Low, Medium, High).
  • Coping Mechanisms: Provide targeted feedback and relaxation tips dynamically.
Slide 2 of 6 • Data Engineering

The Dataset & Feature Space

Data Collection

The system uses a highly structured student behavior and mental wellness dataset comprising records of over 1,100 students. The features encapsulate multi-dimensional parameters of a student's daily lifecycle.

Key Features (20 Parameters)

  • Academic: Study Hours, GPA, Academic Workload.
  • Sleep & Health: Sleep Quality, Sleep Duration, Physical Activity.
  • Psychological: Anxiety Levels, Peer Pressure, Family History.
  • Social: Screen Time, Social Media Usage, Relationship satisfaction.
Slide 3 of 6 • Preprocessing Pipeline

Preprocessing Steps

Feature Pipeline

  • Data Cleaning: Managed missing entries, verified distribution density, and neutralized outliers.
  • Encoding: Converted textual categorical indicators to numeric values via Label Encoding.
  • Feature Scaling: Applied standard scaling to handle varying units (e.g. study hours vs screen time).

Train-Test Splitting

Divided the cleaned dataset into an **80/20 train-test ratio** to perform secure validation.

Slide 4 of 6 • AI / ML Engine

AI / ML Model — Random Forest Classifier

Algorithm Rationale

This project applies an AI / ML approach using a Random Forest Classifier — an ensemble learning model that builds multiple decision trees and combines their results through majority voting for accurate, robust predictions.

Core Model Advantages

  • Overfitting Mitigation: Random sub-sampling of features minimizes individual tree dependency.
  • Handles Non-Linearity: Excellent at finding patterns in interacting variables (e.g., Sleep vs Workload).
  • Feature Importance: Able to output exactly which indicators contribute most to stress.
